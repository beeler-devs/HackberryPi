## 1. Overview

The **MCP4131** is a single-channel, 7-bit (129-step) volatile digital potentiometer manufactured by Microchip Technology. It is controlled over an **SPI-compatible serial interface** and stores the wiper position in RAM only — the wiper resets to mid-scale on power-up. It is pin-compatible with the MCP4141 (non-volatile EEPROM variant) and supports both potentiometer and rheostat configurations.

The MCP4131 is part of the broader MCP41/42XX family covering single and dual configurations, 7-bit and 8-bit resolutions, and volatile (RAM) or non-volatile (EEPROM) memory types.

**Core specifications at a glance:**

| Parameter         | Value                              |
|-------------------|------------------------------------|
| Resolution        | 7-bit (128 resistors, 129 steps)   |
| RAB resistance    | 5 kΩ, 10 kΩ, 50 kΩ, or 100 kΩ    |
| Wiper resistance  | 75 Ω typical                       |
| Supply voltage    | 1.8 V to 5.5 V                     |
| Interface         | SPI (Modes 0,0 and 1,1)            |
| SPI clock speed   | Up to 10 MHz                       |
| Memory            | Volatile (RAM) — mid-scale POR     |
| Power-on reset    | Wiper set to mid-scale (64)        |
| Max wiper current | 2.5 mA per terminal                |

## 2. Key Features

- **Single resistor network** with potentiometer (3-terminal) configuration
- **7-bit resolution:** 128 resistors, 129 addressable wiper positions (0–128)
- **RAB resistance options:** 5 kΩ, 10 kΩ, 50 kΩ, 100 kΩ
- **Zero-scale to full-scale wiper operation**
- **Low wiper resistance:** 75 Ω typical
- **Low temperature coefficient:**
  - Absolute (Rheostat): 50 ppm/°C typical (0°C to 70°C)
  - Ratiometric (Potentiometer): 15 ppm/°C typical
- **SPI serial interface:** 10 MHz, modes 0,0 (CPOL=0, CPHA=0) and 1,1 (CPOL=1, CPHA=1)
- **SDI/SDO pin multiplexing** on 8-pin packages (single-device packages are pin-limited)
- **High-voltage tolerant digital inputs:** up to 12.5 V
- **Shutdown capability:**
  - Hardware: SHDN pin (N/C on 8-lead single packages; use TCON instead)
  - Software: Terminal Control (TCON) register
- **Brown-out reset protection:** ~1.5 V threshold
- **Serial interface inactive current:** 2.5 µA typical
- **Supports split-rail applications**
- **Internal weak pull-up on all digital inputs**
- **Automotive grade versions available**

### 8-Lead PDIP / SOIC / MSOP / DFN Package (Single Potentiometer)

```
         ┌────────┐
   CS ───┤1      8├─── VDD
  SCK ───┤2      7├─── P0B (Terminal B)
SDI/SDO──┤3      6├─── P0W (Wiper)
  VSS ───┤4      5├─── P0A (Terminal A)
         └────────┘
```

| Pin | Name    | Type       | Description                                                                                     |
|-----|---------|------------|-------------------------------------------------------------------------------------------------|
| 1   | CS      | HV, I (ST) | Chip Select — active low; pulling low enables SPI commands                                      |
| 2   | SCK     | HV, I (ST) | Serial Clock — SPI clock input from host controller; idle state depends on SPI mode             |
| 3   | SDI/SDO | HV, I/O    | Serial Data In/Out (multiplexed) — input during command phase; open-drain output during reads   |
| 4   | VSS     | P          | Ground reference for all internal circuitry                                                     |
| 5   | P0A     | A          | Potentiometer Terminal A — connect to one end of the resistive ladder                           |
| 6   | P0W     | A          | Potentiometer Wiper — variable tap output; can source/sink up to 2.5 mA                        |
| 7   | P0B     | A          | Potentiometer Terminal B — connect to the other end of the resistive ladder                     |
| 8   | VDD     | P          | Positive power supply (1.8 V – 5.5 V)                                                          |

**Pin type key:**  
`HV` = High-Voltage tolerant (up to 12.5 V) | `I` = Digital Input | `ST` = Schmitt Trigger | `A` = Analog | `P` = Power | `I/O` = Bidirectional

> **Note on SDI/SDO multiplexing:** The 8-lead single-potentiometer package is pin-limited and combines SDI and SDO onto one pin. After receiving the first 6 bits (address + command), if a Read command was issued, the pin switches to output mode (SDO). The internal pull-up is smart: it disables itself while the pin is driven low, reducing standby current. For faster SDO transitions, an external pull-up resistor can be added.

> **Note on DFN/QFN packages:** These packages have an exposed thermal pad on the bottom of the package that is electrically connected to the die substrate (VSS). It should be left unconnected or tied to the same ground as VSS.

## 9. Resistor Network Architecture

The MCP4131 contains a resistive ladder network made up of 128 equal resistors (RS) in series between Terminal A and Terminal B. Each node between resistors connects via an analog switch to the common wiper bus.

```
P0A ──┬──[RS]──┬──[RS]──┬──[RS]── ... ──┬──[RS]──┬── P0B
      │        │        │               │        │
      SW128   SW127    SW126           SW1      SW0
      │        │        │               │        │
      └────────┴────────┴───────────────┴────────┴── P0W (Wiper)
```

- **Full-scale (wiper = 128):** P0W connected to P0A → RWB = RAB + RW
- **Zero-scale (wiper = 0):** P0W connected to P0B → RWA = RAB + RW
- **Mid-scale (wiper = 64, POR default):** P0W is mid-ladder

The resistance from wiper to terminal B as a function of wiper code (D):

```
R_WB(D) = (D / 128) × RAB + RW
R_WA(D) = ((128 - D) / 128) × RAB + RW
```

Where:
- `D` = wiper register value (0 to 128)
- `RAB` = end-to-end resistance (nominal 5k, 10k, 50k, or 100k Ω)
- `RW` = wiper contact resistance (~75 Ω typical)

**Output voltage in potentiometer mode (divider):**

```
V_W = V_A × (D / 128)    (when V_B = 0)
```

For a 5 V supply with RAB = 10 kΩ and D = 64:
```
R_WB = (64 / 128) × 10000 + 75 = 5075 Ω
V_W  = 5.0 × (64 / 128) = 2.5 V
```

## 10. SPI Interface

### Overview

The MCP4131 operates as an **SPI slave only** — it does not generate the clock. It uses up to 3 pins in standard operation: CS, SCK, and the multiplexed SDI/SDO pin (3-wire SPI).

**Supported SPI modes:**
- Mode 0,0 (CPOL = 0, CPHA = 0): clock idle low, data sampled on rising edge
- Mode 1,1 (CPOL = 1, CPHA = 1): clock idle high, data sampled on falling edge

**MSB first.** All transactions are MSB-first.

### SPI Timing Diagram

```
         ─────────────────────────────────────────────────────
CS        HIGH          │     LOW (active)        │    HIGH
         ─────────────────────────────────────────────────────
              ┌──┐  ┌──┐  ┌──┐  ┌──┐  ┌──┐  ┌──┐  ┌──┐  ┌──┐
SCK      ────┘  └──┘  └──┘  └──┘  └──┘  └──┘  └──┘  └──┘  └────
         
SDI/SDO  ├─ AD3 ─┤─ AD2 ─┤─ AD1 ─┤─ AD0 ─┤─ C1 ──┤─ C0 ──┤─ D9 ──┤─ D8 ──┤
               (Command Byte, MSB first)
                              followed by Data Byte (D7..D0) for write commands
```

### CS Pin Behavior

- `CS = HIGH (VIHH)`: Device deselected. Enables high-voltage serial commands.
- `CS = LOW (VIL)`: Device selected. Normal serial commands accepted.
- CS must return HIGH between independent transactions.
- CS can remain LOW for continuous multi-command sequences (see Section 7.4 of the datasheet).

## 11. Command Set & Memory Map

### Memory Map

| Address | Register          | Supported Commands                                  |
|---------|-------------------|-----------------------------------------------------|
| 00h     | Volatile Wiper 0  | Write Data, Read Data, Increment Wiper, Decrement Wiper |
| 01h     | Volatile Wiper 1  | (reserved for dual-pot devices; N/A on MCP4131)    |
| 02h–03h | Reserved          | —                                                   |
| 04h     | TCON Register     | Write Data, Read Data                               |
| 05h     | STATUS Register   | Read Data only                                      |
| 06h–0Fh | Reserved          | —                                                   |

### Command Byte Format

The first byte sent after CS goes low is the **command byte**:

```
Bit:  7    6    5    4    3    2    1    0
     AD3  AD2  AD1  AD0   C1   C0   D9   D8
```

| Field    | Bits  | Description                                      |
|----------|-------|--------------------------------------------------|
| AD3:AD0  | [7:4] | Register address (see memory map above)          |
| C1:C0    | [3:2] | Command: 00=Write, 11=Read, 01=Increment, 10=Decrement |
| D9:D8    | [1:0] | Most significant 2 bits of data (for write commands) |

### Command Encoding

| C1 | C0 | Command           | Frame Length | Notes                                       |
|----|----|-------------------|--------------|---------------------------------------------|
| 0  | 0  | Write Data        | 16 bits      | CMD byte + Data byte (D7–D0)                |
| 1  | 1  | Read Data         | 16 bits      | CMD byte + dummy byte; device returns data  |
| 0  | 1  | Increment Wiper   | 8 bits       | Single byte; wiper clamps at 128            |
| 1  | 0  | Decrement Wiper   | 8 bits       | Single byte; wiper clamps at 0              |

### Command Byte Examples (for Wiper 0 at address 00h)

| Operation              | Byte 1 (hex) | Byte 2 (hex) | Notes                          |
|------------------------|--------------|--------------|--------------------------------|
| Write wiper = 0        | 0x00         | 0x00         | Set to minimum resistance      |
| Write wiper = 64       | 0x00         | 0x40         | Set to mid-scale               |
| Write wiper = 128      | 0x00         | 0x80         | Set to maximum (D8 bit = 1)    |
| Read wiper value       | 0x0C         | 0x00 (dummy) | 0x0C = 0000 1100               |
| Increment wiper +1     | 0x04         | (none)       | 0x04 = 0000 0100               |
| Decrement wiper −1     | 0x08         | (none)       | 0x08 = 0000 1000               |
| Write TCON register    | 0x40         | 0xFF         | 0x40 = 0100 0000               |
| Read TCON register     | 0x4C         | 0x00 (dummy) | 0x4C = 0100 1100               |

> **Full-scale wiper write (D = 128):** Since 128 = 0x80, bit D8 must be set in the command byte. The command byte becomes `0x01` (address 00h, write command, D9=0, D8=1) and the data byte is `0x00`. Most implementations simply send `0x00, 0x80` which correctly places bit 7 of the data byte as D8=1 via the 9-bit data field concatenation.

### 16-Bit Write Transaction (Arduino-style)

```
CS = LOW
SPI.transfer(0x00);   // Address=0000, Command=00 (write), D9:D8=00
SPI.transfer(value);  // D7:D0 = wiper position (0–127); use 0x80 for 128
CS = HIGH
```

### Continuous Commands (CS held LOW)

While CS remains LOW, any sequence of valid commands may be sent back-to-back without toggling CS. This allows rapid updates to the wiper in time-sensitive applications.

## 12. TCON Register

The **Terminal Control (TCON) register** (address 04h) provides software control over each terminal's connection to the resistor network. This allows soft shutdown without using the hardware SHDN pin.

### TCON Register Bit Map

```
Bit:   7    6    5    4    3    2    1    0
      D8  R1HW  R1A  R1W  R1B  R0HW R0A  R0W  R0B
```

> D8 is a reserved bit forced to 1 on reads.

For the MCP4131 (single pot), only the lower 4 bits (D3:D0) are relevant:

| Bit | Name | Function                                                                |
|-----|------|-------------------------------------------------------------------------|
| D3  | R0HW | Hardware Shutdown Status (read-only): 1 if SHDN pin is HIGH (active)   |
| D2  | R0A  | Terminal A connect: 1 = connected to resistor network; 0 = disconnected |
| D1  | R0W  | Wiper connect: 1 = connected; 0 = disconnected (Hi-Z)                  |
| D0  | R0B  | Terminal B connect: 1 = connected; 0 = disconnected                    |

**Default (all connected):** TCON = 0xFF (all bits = 1)  
**Software shutdown:** Set R0A=0, R0W=0, R0B=0 → TCON = 0xF8 (upper nibble still 1)

When SHDN pin returns HIGH, TCON controls are restored and the wiper register is unaffected.

## 14. Shutdown Operation

The MCP4131 supports two shutdown methods:

### Hardware Shutdown (SHDN Pin)
- Not accessible on 8-lead single packages (pin is NC or internally managed)
- When asserted low, disconnects all three resistor network terminals from the ladder
- Wiper register value is **preserved** during shutdown
- TCON register is **not modified** by the SHDN pin

### Software Shutdown (TCON Register)
The recommended method for 8-lead packages. Set R0A, R0W, R0B bits to 0:

```
// Send write to TCON at address 04h
CS = LOW
SPI.transfer(0x40);   // 0100 0000: address=0100, command=00 (write)
SPI.transfer(0xF8);   // R0HW=1, R0A=0, R0W=0, R0B=0
CS = HIGH
```

To restore normal operation:
```
SPI.transfer(0x40);
SPI.transfer(0xFF);   // All terminals reconnected
```

## 15. Wiper Resistance Formula

```
R_WB(D) = (D / 128) × RAB + RW        (0 ≤ D ≤ 128)
R_WA(D) = ((128 - D) / 128) × RAB + RW

R_WW     = RW ≈ 75 Ω (wiper to wiper, zero-step)
```

**Voltage divider output (V_B = 0, V_A = VDD):**

```
V_W = VDD × (D / 128)
```

**Step size for common RAB values:**

| RAB     | LSB Step (RS = RAB/128) |
|---------|-------------------------|
| 5 kΩ   | ~39 Ω/step              |
| 10 kΩ  | ~78 Ω/step              |
| 50 kΩ  | ~391 Ω/step             |
| 100 kΩ | ~781 Ω/step             |

**Voltage step size (voltage divider mode, VDD = 5 V):**

```
ΔV = VDD / 128 ≈ 39.06 mV/step
```

| 8 (VDD)     | +5V (with 0.1µF bypass) | Power |

> The SDI/SDO multiplexed pin can be connected to both MOSI and MISO. A 10 kΩ resistor in series on the MISO line prevents bus conflicts when the MCU drives MOSI (SDI mode). For write-only applications, only connect SDI to MOSI and skip the MISO connection.

### SPI Configuration

```
SPI Mode:    0 (CPOL=0, CPHA=0)  or  Mode 3 (CPOL=1, CPHA=1)
Bit order:   MSB first
Clock speed: Up to 10 MHz (5 MHz recommended for most breadboard setups)
```

## 19. Schematic: Potentiometer vs. Rheostat Configuration

### Potentiometer (3-terminal voltage divider)

```
VDD ──── P0A ──[ladder]── P0B ──── GND
                   │
                  P0W ───── V_out
```

Used for: adjustable gain, DAC-like voltage generation, sensor calibration.

### Rheostat (2-terminal variable resistor)

```
Signal ──── P0A or P0B ──[ladder]── P0W ──── Output
            (unused terminal left open or tied to P0W)
```

Used for: current limiting, adjustable filter cutoffs, volume/brightness control where load is referenced to a rail.

> For rheostat use, connect P0B to P0W (short them together) and use P0A and P0W as the two resistor terminals. This minimizes wiper resistance error effects.

## 20. Code Examples

### Arduino (C++)

```cpp
#include <SPI.h>

const int CS_PIN = 10;
const byte ADDRESS_WIPER0 = 0x00;  // Write command to Wiper 0

void setup() {
  pinMode(CS_PIN, OUTPUT);
  digitalWrite(CS_PIN, HIGH);
  SPI.begin();
  SPI.setClockDivider(SPI_CLOCK_DIV8);  // ~2 MHz on 16 MHz Arduino
  SPI.setDataMode(SPI_MODE0);           // CPOL=0, CPHA=0
  SPI.setBitOrder(MSBFIRST);
}

void loop() {
  // Sweep wiper from 0 to 128 and back
  for (int i = 0; i <= 128; i++) {
    setWiper(i);
    delay(10);
  }
  for (int i = 128; i >= 0; i--) {
    setWiper(i);
    delay(10);
  }
}

void setWiper(int value) {
  // value: 0 to 128
  // For values 0–127: CMD byte = 0x00, data byte = value
  // For value 128:    CMD byte = 0x01, data byte = 0x00 (D8 bit set)
  byte cmd = (value > 127) ? 0x01 : 0x00;
  byte dat = (value > 127) ? 0x00 : (byte)value;

  digitalWrite(CS_PIN, LOW);
  SPI.transfer(cmd);
  SPI.transfer(dat);
  digitalWrite(CS_PIN, HIGH);
}

void incrementWiper() {
  digitalWrite(CS_PIN, LOW);
  SPI.transfer(0x04);  // Address=00h, Command=01 (increment)
  digitalWrite(CS_PIN, HIGH);
}

void decrementWiper() {
  digitalWrite(CS_PIN, LOW);
  SPI.transfer(0x08);  // Address=00h, Command=10 (decrement)
  digitalWrite(CS_PIN, HIGH);
}
```

```c
void mcp4131_init(void);

// mcp4131.c
#include "mcp4131.h"
#include "stm32f4xx.h"

// CS on PA0, SPI1 on PA5(SCK), PA6(MISO), PA7(MOSI)
#define CS_PORT  GPIOA
#define CS_PIN   GPIO_PIN_0

static void cs_low(void)  { CS_PORT->ODR &= ~CS_PIN; }
static void cs_high(void) { CS_PORT->ODR |=  CS_PIN; }

static uint8_t spi_transfer(uint8_t data) {
  while (!(SPI1->SR & SPI_SR_TXE));  // Wait TX empty
  SPI1->DR = data;
  while (!(SPI1->SR & SPI_SR_RXNE)); // Wait RX full
  return SPI1->DR;
}

void mcp4131_set_value(uint8_t value) {
  uint8_t cmd  = (value > 127) ? 0x01 : 0x00;
  uint8_t data = (value > 127) ? 0x00 : value;
  cs_low();
  spi_transfer(cmd);
  spi_transfer(data);
  cs_high();
}

void mcp4131_increment(void) {
  cs_low();
  spi_transfer(0x04);
  cs_high();
}

void mcp4131_decrement(void) {
  cs_low();
  spi_transfer(0x08);
  cs_high();
}
```

### MicroPython (Raspberry Pi Pico / ESP32)

```python
from machine import Pin, SPI

class MCP4131:
    """7-bit SPI digital potentiometer (MCP4131)."""

    def __init__(self, spi: SPI, cs: Pin):
        self.spi = spi
        self.cs = cs
        self.cs.value(1)
        self._value = 64  # mid-scale default (POR state)

    def set(self, value: int) -> int:
        """Set wiper to value 0–128."""
        value = max(0, min(128, value))
        cmd = 0x01 if value > 127 else 0x00
        dat = 0x00 if value > 127 else value
        self.cs.value(0)
        self.spi.write(bytes([cmd, dat]))
        self.cs.value(1)
        self._value = value
        return self._value

    def get(self) -> int:
        """Return tracked wiper value (reads not easily supported due to muxed pin)."""
        return self._value

    def increment(self) -> int:
        if self._value < 128:
            self.cs.value(0)
            self.spi.write(bytes([0x04]))
            self.cs.value(1)
            self._value += 1
        return self._value

    def decrement(self) -> int:
        if self._value > 0:
            self.cs.value(0)
            self.spi.write(bytes([0x08]))
            self.cs.value(1)
            self._value -= 1
        return self._value

    def fraction(self) -> float:
        """Return wiper position as 0.0–1.0."""
        return self._value / 128.0

# Usage example
spi = SPI(0, baudrate=1_000_000, polarity=0, phase=0)
cs  = Pin(17, Pin.OUT)
pot = MCP4131(spi, cs)
pot.set(64)   # mid-scale
pot.increment()
```

## 21. Design Guidelines & Best Practices

### Power Supply Decoupling
- Place a **0.1 µF ceramic capacitor** as close as possible to the VDD pin (pin 8), bypassing to VSS.
- Optionally add a **10 µF bulk capacitor** on the supply rail for additional filtering.

### SPI Configuration
- Use **SPI Mode 0** (CPOL=0, CPHA=0) or **Mode 3** (CPOL=1, CPHA=1). Mode 0 is most common.
- Keep SPI clock at or below **10 MHz**. For breadboard prototyping, 1–5 MHz is more reliable.
- Use **MSB first** bit ordering.
- The MCP4131 does **not** support Mode 1 or Mode 2.

### Multiple Devices on One SPI Bus
- Assign a unique CS pin per MCP4131 instance.
- CS pins on inactive devices must remain HIGH; only one device's CS should be LOW at any time.

### SDI/SDO Multiplexing
- The 8-pin package combines SDI and SDO. For **read-back**, connect pin 3 to both MOSI and MISO via isolation:
  - Connect MOSI → SDI/SDO directly.
  - Connect MISO → SDI/SDO via a 10 kΩ series resistor or use SPI in half-duplex mode.
- For **write-only** applications, simply connect MOSI to pin 3 and ignore MISO.

### Terminal Loading
- Maximum current through any terminal (P0A, P0B, P0W): **2.5 mA**.
- For output currents above this, buffer the wiper with an op-amp (voltage follower).
- In rheostat mode, the wiper can source/sink current in both directions.

### Wiper Resistance Compensation
- The nominal wiper resistance RW ≈ 75 Ω varies with temperature and supply voltage.
- In precision applications, measure and calibrate RW at runtime.
- For potentiometer (ratiometric) mode, RW cancels out if both ends are driven identically.

### Split-Rail Operation
- The MCP4131 supports split-rail configurations where P0B is below VSS (negative rail).
- Ensure the analog terminal voltages remain within: `VSS − 0.6 V` to `VDD + 0.6 V`.

### ESD Handling
- The device is ESD sensitive. Use proper handling procedures (ESD wrist straps, mats, bags).
- High-voltage tolerant digital inputs (up to 12.5 V) are for logic-level compatibility only, not overvoltage protection on analog terminals.

## 22. Troubleshooting

| Symptom                        | Likely Cause                                                       | Solution                                                                    |
|--------------------------------|--------------------------------------------------------------------|-----------------------------------------------------------------------------|
| No response / wiper stuck      | CS not toggled correctly (must go LOW then HIGH per transaction)   | Verify CS goes LOW before and HIGH after each SPI transaction               |
| Incorrect wiper position       | Wrong wiper value in data byte (> 127 needs D8 bit in cmd byte)   | For D=128, send cmd=0x01, data=0x00 instead of cmd=0x00, data=0x80          |
| Wiper resets to 64 on power-up | Normal behavior: MCP4131 is volatile, POR sets wiper to mid-scale | Expected. Use MCP4141 (EEPROM) if non-volatile wiper position is needed     |
| Read always returns 0xFF       | SDI/SDO multiplexed pin not connected to MISO                      | Connect pin 3 to both MOSI and MISO (with isolation resistor on MISO)       |
| Noise on wiper output          | Missing decoupling capacitor on VDD                                | Add 0.1 µF ceramic cap close to VDD pin                                     |
| SPI clock not working          | Incorrect SPI mode; MCP4131 supports only modes 0,0 and 1,1       | Set CPOL=0, CPHA=0 (Mode 0) or CPOL=1, CPHA=1 (Mode 3)                    |
| Output voltage doesn't reach 0 | Wiper resistance RW (~75 Ω) limits zero-scale output               | Account for RW in calculations; use op-amp buffer if precision is required  |
| Wiper stuck at max or min      | Increment/decrement clamped at boundary (0 or 128)                | Use write command to reset position; device clamps, does not wrap           |
